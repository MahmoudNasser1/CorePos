## 1. نموذج Multi-Tenancy المستخدم

**النموذج المختار: Shared Database + Tenant enforcement في الباك**

```
┌─────────────────────────────────────────────────────────┐
│                  قاعدة بيانات واحدة                     │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ شركة A   │  │ شركة B   │  │ شركة C   │  ...       │
│  │ id: 001  │  │ id: 002  │  │ id: 003  │             │
│  └──────────┘  └──────────┘  └──────────┘             │
│                                                         │
│  كل صف في كل جدول له company_id                        │
│  الباك يمنع رؤية بيانات شركة أخرى (tenant isolation)  │
└─────────────────────────────────────────────────────────┘
```

**لماذا هذا النموذج؟**
- ✅ أسهل في التطوير والصيانة
- ✅ عزل المستأجرين عبر middleware/guards في الباك
- ✅ لا تحتاج قواعد بيانات منفصلة
- ✅ تقارير cross-tenant للـ Super Admin سهلة

## 5. منطق التحقق من الحدود (Middleware)

```typescript
// src/middleware.ts
// قبل كل طلب للـ API، نتحقق من:
// 1. هل المستخدم مسجل؟
// 2. هل اشتراك شركته نشط؟
// 3. هل تجاوز حدود الخطة؟

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // تجاهل الصفحات العامة
  if (pathname.startsWith('/login') ||
      pathname.startsWith('/register') ||
      pathname.startsWith('/pricing')) {
    return NextResponse.next()
  }

  // التحقق من المصادقة
  const user = await getUser(request)
  if (!user) { redirect('/login') }

  // التحقق من الاشتراك (للـ API طلبات الكتابة فقط)
  if (pathname.startsWith('/api/invoices') && request.method === 'POST') {
    const canCreate = await checkInvoiceLimit(user.company_id)
    if (!canCreate) {
      return NextResponse.json(
        { error: 'لقد تجاوزت حد الفواتير الشهري، يرجى الترقية' },
        { status: 403 }
      )
    }
  }
}
```

## 9. تدفق نموذج الفوترة

```
متى تُجدَّد الاشتراكات؟
──────────────────────

Trial (14 يوم):
  اليوم 1  → تسجيل مجاني
  اليوم 12 → تنبيه: "تبقى يومان، جدد اشتراكك"
  اليوم 14 → التطبيق يتوقف (للقراءة فقط)
  اليوم 14+ → يقدر يصدر بياناته، مش يضيف جديد

Monthly:
  كل 30 يوم → Webhook من بوابة الدفع → تحديث current_period_end

Yearly:
  كل 365 يوم → نفس المنطق

Lifetime:
  مدفوع مرة واحدة → ends_at = NULL (لا ينتهي)

Grace Period (فترة السماح):
  بعد انتهاء الدفع بـ 3 أيام → status = 'past_due'
  بعد 7 أيام → status = 'expired' + bقراءة فقط
```
