## 1. الأهداف العامة (تذكير سريع)

| # | الهدف | مؤشر نجاح |
|---|--------|-----------|
| G1 | **اتساق بصري** | نفس `gap`/`padding` ونفس وزن الأزرار لنفس النية |
| G2 | **RTL** | `start`/`end` + جداول وتيم لاين بلا ارتباك |
| G3 | **عربية فقط (واجهة)** | لا لافتة إنجليزية داخل شاشات التطبيق |
| G4 | **أرقام** | Western digits + عملة/تنسيق عبر `formatCurrency` / `Intl` + `latn` حيث ينطبق |
| G5 | **A11y أساسي** | `label`، `aria-label` لأيقونات فقط، تباين مقبول |
| G6 | **طباعة** | A4/80mm: معاية نظيفة، إخفاء chrome |
| G7 | **أداء محسوس** | skeleton، بدون تعليق واضح في POS/التقارير |
| G8 | **سلوك** | بدون `console error` في الرحلة الحرجة (حسب بيئة الاختبار) |

## 2. خريطة: أين ينطبق ماذا؟

| الموضوع | أين تبحث؟ | مخرج |
|---------|-----------|------|
| توكنات/خط | `src/app/globals.css`, `layout.tsx` | تباين، حجم base |
| أزرار/نماذج/جداول | `src/components/ui/*` | سلوك موحّد |
| RTL خاطئ | `grep` لـ `left-` / `ml-` / `right-` في `src` | قائمة إصلاح |
| empty/loading | كل `page.tsx` حرج | لا صفحة «صامتة» |
| طباعة | `*print*`, `globals.css` `@media print` | فاتورة/إيصال |

## 3. التسكات (مفصّلة)

### 3.1 RTL وTailwind — **إلزامي**

- [ ] **T13.1** `grep` استكشافي: `className=.*\b(ml-|mr-|pl-|pr-|left-|right-)\b` في:
  - `src/app/(dashboard)/**`
  - `src/components/**`
- [ ] **T13.2** لكل مورد تنافسي: استبدال بـ `ms`/`me`/`ps`/`pe`/`start`/`end` **حيث يلزم للاتجاه** (لا تغيير عبثي إن كان `left-0` لموضع مطلق مع تصميم معكوس عمدًا — وثّق الاستثناء).
- [ ] **T13.3** **Timeline/جدول زمني:** اختبر على RTL: الخط، النقاط، `border-s`، محاذاة النصوص الثانوية (`text-start` / `text-end` حسب التصميم).
- [ ] **T13.4** **أرقام في الجداول:** فعّل `tabular-nums` للأعمدة المالية (على الأقل: الإجمالي، المدفوع، الرصيد).

### 3.2 دلالات الألوان — **مهم**

- [ ] **T13.5** **مصفوفة تدقيق:** سطر واحد لكل: نجاح، تحذير، خطأ، معلومة، — راجع 3 موديولات (POS، فواتير، خزينة) للتناسق.
- [ ] **T13.6** لا تستخدم `destructive` لنصوص **غير** خطر (مثل رصيد Year-to-date سلبي: قرار بصري — إن كان سلبيًا **مقصودًا** واضحًا، يمكن `muted` + لون مخصّص مقيد بدل مبالغة).

### 3.3 الأيقونات — **مهم**

- [ ] **T13.7** حجم أيقونات buttons صغيرة: `h-4 w-4` داخل `h-8`/`h-9` (أو ما يوازيه في المشروع) — **لا** أيقونات `h-5` داخل `h-7` مزدحمة.
- [ ] **T13.8** `aria-hidden` على أيقونات داخلية عندما النص بجانبها يوضح الإجراء.

### 3.4 الاستجابة (Tablet أولاً) — **إلزامي**

- [ ] **T13.9** عرض `768px` و`1024px` لـ: `/dashboard`, `/dashboard/pos`, `/dashboard/reports/daily` — **لا** تمرير أفقي لـ **نص** (الجداول وحدها تُرخّص).
- [ ] **T13.10** **جداول واسعة:** `overflow-x-auto` + (اختياري) sticky header + عمود تثبيت أول (إن أوضحت الحاجة — UX فقط).
- [ ] **T13.11** **شريط جانبي + محتوى:** في العرض الضيّق، المحتوى لا يُقص بسبب `min-w` لجدول.

### 3.5 التركيز والحوارات — **إلزامي**

- [ ] **T13.12** `Dialog/Sheet/AlertDialog`: **Esc** لإغلاق (سلوك shadcn الافتراضي — تثبيت إن مكسور).
- [ ] **T13.13** **تركيز أولي** في الـ modals: أول حقل إدخال أو زر سلبي `Cancel`؟ اختر **النمط الأقل ضررًا** (غالبًا حقل/زر إلغاء + تركيز على الحقل).
- [ ] **T13.14** **ممنوع** `window.alert` / `window.confirm` للتدفقات الحرجة — استبدل بـ `AlertDialog` / `ConfirmDialog` مع RTL (انظر `src/components/shared/ConfirmDialog.tsx`).

### 3.6 الرسائل — **إلزامي**

- [ ] **T13.15** **مصفوفة رسائل:** لكل موديول، أنواع: شبكة، صلاحية، تعارض بيانات، — جمل **عربية** قصيرة + **ماذا يفعل** المستخدم.
- [ ] **T13.16** `toast` لا يكفي لأخطاء **حقول** — ارفق خطأ **تحت** الحقل.

### 3.7 الطباعة — **إلزامي**

- [ ] **T13.17** `globals.css`: قواعد `@media print` **إن وُجدت** — إخفاء `nav`, `sidebar`, `header` app chrome.
- [ ] **T13.18** اختبر **Print preview** لـ: فاتورة A4، إيصال 80mm — لا **قص** لاسم الشركة/الإجمالي/رقم الفاتورة.

### 3.8 الأداء (واجهة) — **مهم**

- [ ] **T13.19** **Skeleton** للشاشات التي تنتظر API > 300ms: لوحة، تقارير، قوائم الـ 50+ صف.
- [ ] **T13.20** **POS:** لا re-render يسبب وميض إجمالي — استخدم `useMemo`/`useCallback` بـ **مقار** (لا تحسين عبثي لكل مكوّن).

### 3.9 Hydration — **مهم**

- [ ] **T13.21** راجع `suppressHydrationWarning` — يبقى على `html`/`body` فقط عادةً؛ وثّق أي استثناء.

### 3.10 الخاتمة (Pilot) — **إلزامي**

- [ ] **T13.22** حدّث `docs/ui_ux_audit.md` بجداول: عدد 🔴/🟡/🟢 + «قبل/بعد» لمسار واحد.
- [ ] **T13.23** سطر قرار: **UI/UX Ready for Pilot؟** ✅/❌ + 3 أسباب كحد أقصى.

## 4. معايير القبول (Pilot — من Agent-11)

- لا **🔴** مفتوحة.
- **رحلة حرجة:** دخول → لوحة → POS (بيع) → فاتورة/قائمة → تقرير يومي — **بدون** أخطاء console حرجة.
- **POS + طباعة** كما فوق.
- **عربي فقط** في الواجهة؛ **empty states** مفهومة.
