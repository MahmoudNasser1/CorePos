# خطة UI/UX — المخزون (Inventory)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **الملء والقراءة اليومية**: جداول **قابلة المس**، فلاتر **قليلة** وفعّالة، **حذف** دائمًا مع تأكيد بصياغة واضحة.

---

## 1. نظرة عامة

| المسار | الغرض |
|--------|--------|
| `/dashboard/inventory` | ملخص/دخول (إن وُجد) |
| `/dashboard/inventory/products` | القائمة الرئيسية |
| `/new`, `/[id]`, `/[id]/edit` | إضافة/تفاصيل/تعديل |
| `/categories`, `/units` | بيانات مساندة |

**مكوّنات:** `ProductColumns`, `ProductForm`, `BarcodePrintDialog`, (مستقبلاً) حوار/قالب **ملصق منتج (سنيكر)**, جداول.

---

## 2. فلسفة «احترافي + بسيط»

- **قائمة المنتجات = أداة عمل** لا معرض: أعمدة **أولوية** (اسم، سعر بيع، مخزون)؛ الباقي في تفاصيل.
- **تنبيه المخزون:** نفس **المنطق البصري** في القائمة وفي تفاصيل المنتج (تجنّب «منخفض» في مكان و«حرج» في آخر بلا مبرر).
- **نماذج طويلة** → اقسام `Card` بعنوان فرعي — لا صفحة طويلة بلا فواصل.

---

## 3. تدفقات

1. **إضافة منتج:** نموذج → حفظ → **toast** + توجيه لقائمة أو تفاصيل (واحد فقط).
2. **تعديل سريع:** من القائمة → تعديل → **نفس** النمط كـ «جديد».
3. **حذف:** تأكيد → نجاح → الصف **يختفي** أو يُسجّل كمخفي — **واحد** من السلوكين.
4. **باركود:** حوار طباعة → معاية → طباعة.

| # | الهدف | مؤشر |
|---|--------|------|

## 5. التسكات (مفصّلة)

### 5.1 قائمة المنتجات — **إلزامي**

- [ ] **T5.1** **شريط أعلى:** بحث + (فئة/حالة) — إن وُجدت فلترتان فقط في الصف الأول؛ **لا** 8 فلاتر في صف واحد.
- [ ] **T5.2** **جدول:** `sticky` header عند تمرير طويل؛ `overflow-x-auto` مع حد أدنى لعرض الأعمدة.
- [ ] **T5.3** **أعمدة:** اسم+باركود فرعي `text-xs text-muted-foreground`؛ مبالغ `tabular-nums`.
- [ ] **T5.4** **مخزون:** شارة: `secondary` عادي / `destructive` منخفض — **عُدّ** العتبة في التوثيق (مثال ≤5) لتبقى UI متسقًا.
- [ ] **T5.5** **إجراءات (actions):** قائمة `…` — `aria-label` «فتح عمليات المنتج».
- [ ] **T5.6** **فارغ:** «لا توجد منتجات» + `Button` «إضافة منتج».

### 5.2 النموذج (جديد/تعديل) — **إلزامي**

- [ ] **T5.7** **أقسام:** «البيانات الأساسية»، «الأسعار»، «المخزون/الحدود» — عناوين `CardTitle` صغيرة.
- [ ] **T5.8** **حقول رقمية:** `inputMode="decimal"` إن وُفّق؛ تنسيق عرض **بعد** الضبط (لا 10 أرقام عشوائية).
- [ ] **T5.9** **رفع صورة:** حالة **رفع** + معاية مصغّرة — خطأ: «تعذّر رفع الملف. جرّب صورة أصغر».
- [ ] **T5.10** **حفظ:** زر `primary` واحد في أسفل النموذج؛ «جاري الحفظ…».

### 5.3 التفاصيل `products/[id]` — **إلزامي**

- [ ] **T5.11** **رأس:** اسم + فئة/وحدة + شارة **مخزون منخفض** (إن وُجد) — `h1` واحد.
- [ ] **T5.12** **بطاقات:** KPIs في صف — لا تتباعد `space-y-8` بلا داعٍ.
- [ ] **T5.13** **Timeline/مبيعات أخيرة:** تاريخ عربي بـ `date-fns` + `ar` — محاذاة RTL.
- [ ] **T5.14** **تسميات عربية:** مثل «حد الطلب الأدنى» — دون اختلاط لغة (كما تقرر في واجهة المنتج).

### 5.4 التعديل `edit` — **إلزامي**

- [ ] **T5.15** **نفس** `ProductForm` مع `initialData` — لا نسختان مختلفتان بصريًا.
- [ ] **T5.16** **غير موجود:** نفس **حالة 404** كتفاصيل: أيقونة + نص + رابط «العودة للقائمة».

### 5.5 الفئات والوحدات — **مهم**

- [ ] **T5.17** **شجرة/قائمة:** مسافة بادئة `pr-4` لكل مستوى — **لا** خطوط ملتوية إلا إن كانت مقروءة.
- [ ] **T5.18** **حذف فئة/وحدة:** `AlertDialog` — «قد يتأثر تصنيف المنتجات» (صياغة تبع **سياسة المنتج** — UX فقط).

### 5.6 طباعة الباركود — **مهم**

- [ ] **T5.19** `BarcodePrintDialog`: معاية + عدد نسخ + **زر طباعة** واحد.
- [ ] **T5.20** اختفاء عناصر UI غير الضرورية في `print` داخل الحوار/الصفحة.

### 5.7 ملصق منتج «سنيكر» (عرض / بطاقة) — **مهم**

ملصق للرف أو الصندوق: **رمز قابل للمسح** (QR يوجّه إلى SKU أو رابط داخلي للمنتج، أو **باركود خطي** حسب اختيار المستخدم) + **بيانات المنتج** (اسم، SKU، باركود نصي، سعر بيع، وحدة، فئة اختيارية، شعار الشركة إن وُجد في الإعدادات). أسلوب بصري **بطاقة عرض** (typography واضحة، حدود خفيفة، `tabular-nums` للأسعار) — منفصل عن «نسخ باركود متكررة» في 5.6.

- [ ] **T5.21** **مدخل من القائمة/التفاصيل:** إجراء «طباعة ملصق منتج» بجانب «طباعة الباركود» — يفتح حوارًا جديدًا أو تبويبًا داخل حوار موحّد (`ProductLabelPrintDialog`) مع خيار **نوع الرمز:** QR | باركود خطي (نفس قيمة `barcode`/`sku` المعتمدة في المخزون).
- [ ] **T5.22** **حقول قابلة للتبديل** في المعاينة (إظهار/إخفاء): السعر، الفئة، الشعار — الافتراضي: اسم + SKU + رمز + سعر + وحدة.
- [ ] **T5.23** **مقاس ورق:** قالب طباعة واحد على الأقل (مثال 62×100 مم أو A4 متعدد الملصقات) + عدد نسخ؛ نفس قواعد `@media print` وإخفاء أزرار الواجهة كـ T5.20.
- [ ] **T5.24** توليد QR من مكتبة موثوقة (مثلاً `qrcode` أو ما يتوافق مع المشروع) — المحتوى: نص ثابت قابل للتوسعة (SKU أو `app URL + product id` حسب قرار المنتج).
- [ ] **T5.25** **اتساق مع الطباعة العامة:** توثيق المسار في `docs/reports/printing-system-review.md` (قسم إضافات) عند التنفيذ.
- [ ] **T5.26** **الضريبة على الملصق (إن طُلب السعر شاملاً):** لا تعرض **نسبة** ضريبة ثابتة؛ إما سعر بيع فقط كما في المنتج، أو سطر «شامل الضريبة» نصيًا حسب إعداد الشركة — **بدون** رقم نسبة افتراضي في الواجهة.

## 6. نسخ جاهز

| السياق | نص |
|--------|-----|
| حذف منتج (تأكيد) | «سيتم حذف المنتج ولن يظهر في القوائم. المتابعة؟» |
| نجاح حفظ | «تم حفظ المنتج» |
| لا منتجات | «لا توجد منتجات بعد. ابدأ بإضافة أول صنف.» |
