# خطة UI/UX — العملاء والموردين (Contacts)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **نفس الجودة** لقائمة العميل وقائمة المورد: اختلاف **اللغة** (عميل/مورد) و**اللون** الخفيف فقط.

---

## 1. نظرة عامة

| الكيان | مسارات |
|--------|--------|
| عملاء | `/dashboard/customers`, `/customers/[id]` (كشف/statement إن وُجد) |
| موردون | `/dashboard/suppliers`, `/suppliers/[id]` |

**مكوّنات:** `PartnerTable`, `PartnerStatement` (أو ما يعادلها).

**الوظيفة الذهنية:** **أرصدة** واضحة + **رابط** لسريع (فاتورة، سند) — **لا** تفاصيل تسويق.

---

## 2. فلسفة «احترافي + بسيط»

- **قائمة جهات = جدول خفيف** — الإجراءات **في قائمة** `…` لمنع 5 أعمدة أزرار.
- **تفاصيل = بطاقتان:** «البيانات» + «الرصيد/الكشف» — **لا** تداخل بينهما.
- **كشف حساب:** **زمني** (من تاريخ) — فلاتر **في صف** واحد.

---

## 3. تدفقات

1. **قائمة** → **بحث/فلتر** → **صف** → **تفاصيل**.
2. **تفاصيل** → **كشف** (subpage أو تبويب) — **نفس** أسلوب `finance` للتمرير/الفلتر.
3. **عميل جديد** (إن وُجد) — **نموذج قصير** + «حفظ وإغلاق».

---

## 5. التسكات (مفصّلة)

### 5.1 القوائم — **إلزامي**

- [ ] **T9.1** **أعمدة موحّدة** بين العملاء/الموردين: اسم، هاتف، **رصيد**، **إجراءات** (أو CTA ثابت).
- [ ] **T9.2** **رصيد:** لون `destructive` **فقط** لديون واضح — رصيد صفر/موجب `muted`/`default` (قرار product).
- [ ] **T9.3** **بحث:** debounce؛ placeholder عربي «ابحث بالاسم أو الهاتف…».
- [ ] **T9.4** **فارغ:** «لا عملاء بعد» + «إضافة عميل» (إن وُجد المسار).

### 5.2 التفاصيل — **إلزامي**

- [ ] **T9.5** **رأس:** اسم + `Badge` (نشط/موقوف إن وُجد) + أزرار: «تعديل»، **«كشف حساب»** كـ `outline`.
- [ ] **T9.6** **بطاقة ملخص** رصيد: رقم `text-3xl` **واحد** + تسمية «الرصيد الحالي».
- [ ] **T9.7** **روابط سريعة** (إن وُجد): «فاتورة جديدة» / «سند قبض» — `Button` + أيقونات `h-4 w-4`.

### 5.3 كشف الحساب / Statement — **إلزامي**

- [ ] **T9.8** **فلتر** من-إلى + «تطبيق» + «تصفير».
- [ ] **T9.9** **جدول:** `sticky` header؛ `tabular-nums`؛ **لا** تتباعد مبالغ.
- [ ] **T9.10** **طباعة/تصدير** (إن وُجد): **معاية** + `@media print` — كما `InvoicePrint`.

### 5.4 الاتساق — **مهم**

- [ ] **T9.11** **diff audit:** افتح عميل + مورد جنبًا إلى جنب — اختلاف **لون الشارة** + **نص** فقط، لا اختلاف `padding` بلا مبرر.
