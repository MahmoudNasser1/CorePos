# خطة UI/UX — الإعدادات + التسويق + الفوترة (Settings / Marketing / Billing)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **إعدادات = هدوء**، ألوان `muted` أكثر، **بلا** مبالغة؛ **تسويق = طاقة** لكن **نفس** هوية المنتج (Cairo, RTL, spacing).

---

## 1. نظرة عامة

| المجال | مسارات (تقريبية) |
|--------|------------------|
| إعدادات | `/dashboard/settings`, `.../branches`, `.../warehouses`, `.../invoice` |
| تسويق | `(marketing)/page.tsx` — صفحة هبوط |
| فوترة | `(billing)/billing`, `.../history`, `.../upgrade`, `.../expired` |

**ملاحظة:** `docs/screens_map.md` قد يسرد إعدادات إضافية (مستخدمون، ضرائب…) — **ادمجها** عند توسيع نفس الملف لاحقًا.

---

## 2. فلسفة «احترافي + بسيط»

- **الإعدادات:** قائمة يسرى أو تبويبات **ثابتة** — **لا** تدفق معقد من 6 مستويات.
- **تغيير يؤثر على الفواتير/الشركة** → `AlertDialog` **قبل** الحفظ (UX فقط).
- **صفحة تسويق:** أقسام **كبيرة** (Hero → ميزات → أسعار) — **3–5** كتل، لا 20 قسمًا.

---

## 3. تدفقات

1. **الإعدادات** → اختيار بند (فرع/مخزن) → **نموذج** → **حفظ** → toast.
2. **الفوترة** → **حالة** الخطة (نشط/منتهٍ) → CTA.
3. **تسويق** → CTA **تسجيل/دخول** — اختبر على موبايل (أغلب المستخدمين هناك).

---

## 4. الأهداف

| # | الهدف | مؤشر |
|---|--------|------|
| G1 | تنقل إعدادات | تبويبات واضحة |
| G2 | فروع/مخازن | CRUD + تأكيد |
| G3 | فاتورة/شعار | رفع + معاية |
| G4 | تسويق | CTA + ثقة بصرية |
| G5 | فوترة | وضوح + سجل + expired |

## 5. التسكات (مفصّلة)

### 5.1 هيكل الإعدادات — **إلزامي**

- [ ] **T11.1** `SettingsNav` (أو قائمة يسرى): ارتفاع `min-h` للمحتوى — العنوان النشط `bg-accent`.
- [ ] **T11.2** **صفحات** فرعية: **نفس** `max-w-4xl` مثلًا — لا صفحات **ضيقة** وأخرى **واسعة** بلا مبرر.
- [ ] **T11.3** **عنوان** كل صفحة: «الفروع»، «المخازن» — `h1` + وصف `text-sm` (اختياري).

### 5.2 الفروع والمخازن — **إلزامي**

- [ ] **T11.4** **جدول/بطائق:** اختيار **واحد** (جدول لكثير من الصفوف، بطائق لعدد قليل).
- [ ] **T11.5** **نموذج** إضافة: أقل حقول **ضرورية**؛ `description` اختياري مموّل بـ `text-muted-foreground` تحت الحقل.
- [ ] **T11.6** **حذف** فرع/مخزن — `AlertDialog` وشرح: «لن تُحذف الفواتير…» (حسب product).

### 5.3 إعدادات الفاتورة — **مهم**

- [ ] **T11.7** `InvoiceSettingsForm`: **Logo** — معاية `object-contain` + حد أقصى ارتفاع.
- [ ] **T11.8** **معاية قالب** (إن وُجد) — A4 **مصغّر** في كرت.
- [ ] **T11.9** **RTL** في المعاية — اختبر مع اسم شركة طويل.

### 5.4 صفحة التسويق — **مهم**

- [ ] **T11.10** **Hero:** عنوان **واحد** قوي + subhead سطر + **CTA** `default` (تسجيل) + `outline` (تجربة/دخول).
- [ ] **T11.11** **ميزات:** 3–6 **بطاقات** `Card` **متساوية** الارتفاع تقريبًا.
- [ ] **T11.12** **أسعار:** 2–3 مستوى — اختلاف `border-primary` لما هو «موصى به» — **نص** عربي فقط.
- [ ] **T11.13** **Footer** — روابط/حقوق (عربي).
- [ ] **T11.14** **أداء:** صور `next/image` أو `loading=lazy` حسب المشروع.

### 5.5 الفوترة — **مهم**

- [ ] **T11.15** صفحة `billing`: **بطاقة** «خطتك الحالية» + «تاريخ التجديد/الحد» (حسب product).
- [ ] **T11.16** مسار `billing/upgrade`: **مقارنة** واضحة — لا 10 ميزات بلا `check` مُنظَم.
- [ ] **T11.17** مسار `billing/history`: جدول بسيط، empty state: «لا سجل دفعات بعد».
- [ ] **T11.18** مسار `billing/expired`: **CTA** واحد `primary` + نص: «للمتابعة، تواصل/حدث خطتك» — `destructive` فقط **للحالة** لا للزر إن تسبب إرباكًا.
