# خطط مراجعة UI/UX — موديول موديول (CorePOS)

> **الغرض:** دليل تنفيذي لمراجعة الواجهة **موديولًا بعد موديول**، مع **أهداف واضحة** و**تسكات قابلة للتتبع** و**معايير قبول** لكل موديول.  
> **المرجع العملي:** `.agents/agent-11-ui-ux-audit.md` + `docs/screens_map.md` + `docs/CONTEXT.md`.

---

## 0. مبادئ الواجهة: احترافي + بسيط (اقرأها أولًا)

- **الملف:** [00-design-principles.md](./00-design-principles.md)  
  يتضمّن: **هرمية المحتوى** (عنوان → إجراء → تفاصيل)، **تباعدًا موحّدًا**، **ألوانًا دلالية**، **الحالات الأربع** (تحميل/فارغ/خطأ/نجاح)، **RTL**، **Microcopy** عربي، و**Checklist** قبل الدمج.
- **قاعدة المنتج:** واجهة **لا تزيد تعقيد المستخدم** — كل شاشة تخبره **أين هو** و**ماذا يفعل بعد** في أقل جمل ممكنة.

بعدها انتقل لملف الموديول المطلوب (الجدول أدناه) وطبّق نفس المبادئ على **مسارات وحقول** ذلك القسم.

## 1. المنهجية (قبل أي موديول)

1. **قراءة سريعة:** [00-design-principles.md](./00-design-principles.md) ثم الموديول: المسارات في `src/app` + المكونات في `src/components` ذات الصلة.
2. **تطبيق منظورات المراجعة** (بدون توسيع نطاق business logic):
   - **Nielsen (10) + POS-specific:** وضوح، تطابق النموذج الذهني، التحكم، الاتساق، منع الأخطاء، إظهار الحالة، مرونة الاستخدام، جمال بسيط، مساعدة التعافي، مساعدة التوثيق.
   - **WCAG 2.1 AA (خفيف):** تباين، تسميات، ترتيب تركيز، أدوار/aria حيث يلزم.
   - **RTL أولاً:** `start`/`end`، اتجاه النص، محاذاة الجداول والنماذج.
   - **العربية فقط في الواجهة:** لا نصوص إنجليزية مرئية للمستخدم (metadata/SEO خارج نطاق الشاشة الداخلية).
   - **الأرقام:** غربية + عملة/تنسيق عبر `formatCurrency` / `Intl` بشكل متسق مع `CONTEXT.md`.
   - **الطباعة:** `@media print` للفواتير/الإيصالات (80mm / A4) حيث ينطبق.
   - **الأداء (واجهة):** حالات تحميل، تقليل إعادة الرسم غير الضرورية، تجنب فراغ أثناء الجلب.

3. **توثيق النتائج:** كل مشكلة في `docs/ui_ux_audit.md` بالقالب المحدد في Agent-11 (شدة، موقع، حالي، متوقع، دليل، حالة الإصلاح).
4. **تحديث التقدم:** `docs/agent_reports/PROGRESS.md` بعد كل دفعة إنجاز.
5. **Handoffs:** أي خلل بيانات/API/Auth → `docs/agent_reports/HANDOFFS.md` مع مالك مقترح وخطوات إعادة إنتاج.

## 2. المهارات/المراجع المذكورة في Agent-11 (تُستخدم كـ checklist)

> هذه قائمة **مرجعية** من Agent-11؛ طبّقها عمليًا أثناء كل موديول (لا حاجة لتثبيت skill خارجي إن لم يكن موجودًا — المهم **المخرجات**).

| المرجع | الاستخدام في المراجعة |
|--------|------------------------|
| مراجعة هيوريستيكية (Nielsen + POS) | تدفق البيع، سرعة الإدخال، منع الأخطاء المكلفة |
| إمكانية الوصول (A11y) | حقول النماذج، الجداول، الحوارات، التركيز |
| أداء الواجهة | Skeleton، تجنب lag في POS والجداول الكبيرة |
| أنماط React | state/hooks، منع double submit، معالجة أخطاء واجهة |
| Tailwind + shadcn | اتساق spacing، dark mode إن وُجد، مكونات قابلة للوصول |
| RTL / i18n | اتجاه، فواصل، أيقونات معكوسة منطقيًا |
| طباعة CSS | إخفاء الهيدر/السايدبار عند الطباعة، page breaks |
| POS UX | اختصارات، بحث، سلة، دفع، إيصال |

## 5. تعريف «إغلاق موديول»

- لا توجد 🔴 **blockers** مفتوحة في `docs/ui_ux_audit.md` لهذا الموديول.
- تم تنفيذ التسكات الإلزامية (مميزة بـ **إلزامي**) أو تم تأجيلها بتعليل في `RISKS.md` / `HANDOFFS.md`.
- `npm run lint` + `npx tsc --noEmit` بعد دفعة التعديلات.
- عند الإمكان: تشغيل جزء من `npm run test:e2e` يغطي المسارات الحرجة للموديول.
