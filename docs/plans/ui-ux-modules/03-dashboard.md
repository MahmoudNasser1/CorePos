# خطة UI/UX — لوحة التحكم الرئيسية (Dashboard)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **ملخص سريع في الأعلى**، **تفاصيل في الأسفل**؛ البطاقات **متساوية الارتفاع تقريبًا** ضمن نفس الصف.

---

## 1. نظرة عامة

**المسار:** `src/app/(dashboard)/dashboard/page.tsx`  
**مكوّنات:** `KPIGrid`, `SalesChart`, `TopProductsChart`, `RecentInvoices`, `LowStockWidget`, `StockAlertsWidget` — حسب المشروع الفعلي.

**الوظيفة الذهنية للمستخدم:** «ماذا حدث اليوم؟ ماذا يحتاج انتباه؟» — **ليس** استبدال تقارير مفصّلة.

---

## 2. فلسفة «احترافي + بسيط»

- **3–4 بطاقات رقمية كحد أعلى** في الصف الأول — زيادتها تُشتت دون مبرر.
- **رسم بياني واحد رئيسي** (مبيعات/اتجاه) + باقي تفصيلي أصغر — لا 5 charts بنفس الوزن البصري.
- **نص مساعد (`description`)** جملة واحدة تحت العنوان فقط.

---

## 3. تدفقات

1. **فتح الصفحة:** تظهر أرقام/إطارات خلال **تحميل** (skeleton) لا قفز من 0 إلى قيمة بعد ثانيتين بلا مؤشر.
2. **نقرة من «آخر الفواتير»:** تذهب لتفاصيل فاتورة — زر **رجوع** واضح.
3. **مخزون منخفض:** نقرة → قائمة المنتجات/تفاصيل صنف.

---

## 4. الأهداف

| # | الهدف | مؤشر نجاح |
|---|--------|-----------|
| G1 | KPIs مفهومة | عنوان + رقم + وحدة/عملة + تلميح قصير |
| G2 | charts قابلة للقراءة | تسميات، فراغ empty |
| G3 | responsive | 1 عمود موبايل → 2–4 أعمدة واسعة |
| G4 | لا **بيانات وهمية** بلا توضيح | إن بقي placeholder — شارة «تجريبي» أو handoff |

---

## 5. التسكات (مفصّلة)

### 5.1 الشبكة والبطاقات — **إلزامي**

- [ ] **T3.1** **Grid:** `gap-4` أو `gap-6` موحّد؛ **لا** بطاقة ب`shadow-lg` والبقية `shadow-sm` بلا قصد.
- [ ] **T3.2** بطاقة KPI (`StatCard`): العنوان `text-sm text-muted-foreground`، الرقم `text-2xl font-bold`، `tabular-nums` للمبالغ.
- [ ] **T3.3** **trend (إن وُجد):** سهم/نص «↑ 12%» — لون `success` **فقط** إن المعنى واضح للمستخدم.

### 5.2 الرسوم — **إلزامي**

- [ ] **T3.4** `SalesChart`: محور تاريخ/مبلغ — تداخل التسميات؟ راجع زاوية أو تقليل `tick`.
- [ ] **T3.5** **Empty:** «لا بيانات كافية لهذه الفترة» + اقتراح تغيير نطاق التاريخ (إن وُجد فلتر).
- [ ] **T3.6** `TopProducts`: ترتيب واضح (1،2،3) أو جدول مصغّر **بدون** ألوان مفرطة.

### 5.3 القوائم والتنبيهات — **مهم**

- [ ] **T3.7** `RecentInvoices`: أعمدة: رقم، تاريخ، مبلغ، حالة — الصف **قابلة للنقر** بالكامل (أو رابط واضح).
- [ ] **T3.8** عناصر تنبيه المخزون (`LowStock` / `Alerts`): لون **تحذيري واحد** للمخزون — لا يختلط مع فشل/نجاح.
- [ ] **T3.9** **روابط:** `variant="link"` أو زر `outline` — «عرض الكل →» (سهم **في اتجاه RTL** بصريًا يعني تقدمًا).

### 5.4 الأداء — **مهم**

- [ ] **T3.10** **Skeleton** لكل كرت/رسم عند `loading`.
- [ ] **T3.11** **تجميع طلبات** الـ API إن كان هناك وميض متعدد — (handoff لمن ياملك data layer إن لزم — لا تغيير منطق من هنا).

## 6. نسخ جاهز

| الحالة | نص |
