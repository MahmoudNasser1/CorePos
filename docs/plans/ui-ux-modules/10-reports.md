# خطة UI/UX — التقارير (Reports)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **قراءة طويلة نسبيًا** لكن **غير** مزدحمة: فلتر **أوضح** من الرسم — المستخدم **يعرف البُعد** (يوم/شهر) قبل «تحليل» الرسم.

---

## 1. نظرة عامة

**فهرس:** `/dashboard/reports`  
**تقارير (أمثلة):** `daily`, `sales`, `profits`, `profit-loss`, `stock`, `inventory`, `stock-movement`, `customers`, `suppliers`, `treasury`, `expenses`, `categories`, `tax` — حسب المشروع.

**مكوّنات:** `ReportFilters`, `ReportTable`, `ReportCharts` (تقريبية).

---

## 2. فلسفة «احترافي + بسيط»

- **فهرس التقارير = بطاقات** في شبكة: عنوان + سطر **ماذا يعطيك** (جملة واحدة) + «فتح».
- **تقرير واحد = عنوان + فلتر (ثابت في الأعلى) + نتائج (جدول/رسم)** — **لا** تداخل بين تقريرين في نفس الصفحة.
- **تصدير/طباعة:** **ثانوية** (`outline`/`ghost`) — النتائج أولًا.

---

## 3. تدفقات

1. **فهرس** → اختيار تقرير → **فلتر** افتراضي (مثلاً «هذا الشهر») → **نتائج** فور إمكان.
2. **تغيير تاريخ** → «تطبيق» → **skeleton** للجدول/الرسم **لا** يختفي العنوان.
3. **تصدير** (إن وُجد) → `pending` + «جاري تجهيز الملف…».

---

## 4. الأهداف

| # | الهدف | مؤشر |
|---|--------|------|
| G1 | فهرس منطقي | تجميع حسب: مبيعات/مخزون/مالية/ضرائب |
| G2 | فلتر | presets + مترجم عربي |
| G3 | جداول كبيرة | تمرير/ترقيم/skeleton |
| G4 | رسوم | تسميات غير متداخلة + empty |
| G5 | Gate | تقرير يومي = حد أدنى لدليل الجودة |

## 5. التسكات (مفصّلة)

### 5.1 صفحة الفهرس — **إلزامي**

- [ ] **T10.1** **تجميع:** عناوين أقسام `text-sm font-medium text-muted-foreground` + بطاقات تحتها.
- [ ] **T10.2** **بطاقة:** عنوان التقرير `font-semibold` + وصف `text-sm text-muted-foreground` (سطر واحد).
- [ ] **T10.3** **روابط مكسورة/placeholder:** إن تقرير غير جاهز — **بطاقة** بشارة **«قريبًا»** + `disabled` — أو إخفاء (قرار product).
- [ ] **T10.4** **بحث** في الفهرس (إن وُجد): يساعد **فقط** إن > 12 تقريرًا.

### 5.2 قالب صفحة تقرير (مشترك) — **إلزامي**

- [ ] **T10.5** **تخطيط:** `space-y-6` — 1) عنوان `h1` 2) `ReportFilters` 3) ملخص (اختياري) 4) `ReportTable`/`ReportCharts`.
- [ ] **T10.6** العنوان الرئيسي (`h1`): **اسم** التقرير فقط — الفترة **تحت** في `ReportFilters` كنص: «من … إلى …».

### 5.3 الفلاتر — **إلزامي**

- [ ] **T10.7** `ReportFilters`: `DatePicker` أو نوعي تاريخ (من-إلى) + **أزرار:** «اليوم»، «هذا الأسبوع»، «هذا الشهر».
- [ ] **T10.8** **تطبيق** زر `primary`؛ **تصفية** = **إعادة** لضبط الفلاتر.
- [ ] **T10.9** **خطأ تاريخ** — «التاريخ "من" يجب أن يكون قبل "إلى"».
- [ ] **T10.10** `dir="rtl"` داخل `Popover/Calendar` (تحقق من shadcn).

### 5.4 الجداول — **إلزامي**

- [ ] **T10.11** `ReportTable`: `sticky` header؛ `overflow-x-auto`؛ `tabular-nums` للأرقام.
- [ ] **T10.12** **ترقيم/صفحات** (إن وُجد 1000+ سطر) — اختيار «عدد الصفوف» 25/50/100.
- [ ] **T10.13** **تصدير** — زر + حالة `loading`؛ **لا** تجميد الـ tab.

### 5.5 الرسوم — **مهم**

- [ ] **T10.14** `ReportCharts`: `height` ثابت للرسم (مثال `h-[280px]`) — لا ارتفاع مفاجئ.
- [ ] **T10.15** **تسمية محور** عربي؛ `tick` **لا** يتكدس (rotate -45 فقط عند الضرورة).
- [ ] **T10.16** **Empty:** «لا بيانات للفلتر المختار — غيّر نطاق التاريخ».

### 5.6 التقرير اليومي (Gate) — **إلزامي**

- [ ] **T10.17** تقرير `reports/daily`: يُعد **لقطة** واضحة ليوم التشغيل: إجماليات + حركات — **لا** حشو بلا فائدة.
- [ ] **T10.18** **دليل** في `ui_ux_audit` — لقطة/ملاحظة Lighthouse (اختياري).
