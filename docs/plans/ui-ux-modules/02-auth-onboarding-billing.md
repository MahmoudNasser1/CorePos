# خطة UI/UX — المصادقة + الإعداد الأولي + انتهاء الاشتراك

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **نمط واحد لكل نموذج** (Label فوق، خطأ تحت، زر واضح، بدون تشتيت).

---

## 1. نظرة عامة

| المجال | مسارات تقريبية |
|--------|----------------|
| دخول | `(auth)/login/page.tsx` |
| تسجيل | `(auth)/register/page.tsx` |
| نسيت | `(auth)/forgot-password/page.tsx` |
| إعداد | `(onboarding)/company`, `warehouse`, `sample-data` |
| فوترة / انتهاء | `(billing)/billing/*`, `billing/expired` |

**مبدأ بساطة:** شاشة واحدة = **مهمة واحدة** (مثال: تسجيل دخول لا تخلطه بعرض ميزات تسويق داخل الـ form).

---

## 2. فلسفة «احترافي + بسيط»

- **Form = عمودي، عرض محدود:** `max-w-md`–`max-w-lg` لمركزية النموذج — لا تمتد الحقول بعرض الشاشة إلا لسبب قوي.
- **خطأ = بجانب الحقل أولاً:** الـ `toast` للنجاح/فشل العامة فقط، لا تغني عن رسالة ميدانية.
- **Onboarding = خطوات مرئية:** «الخطوة 2 من 3» + أسماء الخطوات (شركة / مخزن / بيانات) — **بدون** تسميات فنية.

---

## 3. تدفقات حرجة

1. **تسجيل دخول ناجح** → يوجّه بسلاسة للوحة/الإعداد (حسب تطبيقكم) **بدون** وميض.
2. **بيانات خاطئة** → رسالة في الحقل أو تحت النموذج + لا يُعاد الضغط بلا `disabled` على الزر.
3. **Onboarding** → «رجوع» لا يحذف إدخال المستخدم (إن كان محليًا) أو يُنذّر بوضوح.
4. **انتهى الاشتراك** → **لا** يدخل لباقي التطبيق — صفحة `expired` بشرح + CTA واحد.

---

## 4. الأهداف

| # | الهدف | مؤشر نجاح |
|---|--------|-----------|
| G1 | تحقق عربي واضح | كل حقل: رسالة مرتبطة |
| G2 | أثناء الإرسال | `disabled` + نص «جاري…» |
| G3 | فشل الشبكة/401 | نصي عام + لا تسريب بيانات |
| G4 | كلمات المرور | زر عرض/إخفاء + `autocomplete` مناسب |
| G5 | Onboarding | مؤشر تقدم + رجوع آمن |
| G6 | فوترة | وضوح الحالة (نشط/منتهٍ) + CTA التالي |

## 5. التسكات (مفصّلة)

### 5.1 تسجيل الدخول — **إلزامي**

- [ ] **T2.1** **تسميات عربية** لكل حقل: «البريد الإلكتروني أو اسم المستخدم» / «كلمة المرور» — لا مصطلحات API.
- [ ] **T2.2** **Validation:** فاضي/قصير/صيغة — تحت الحقل: «أدخل البريد الإلكتروني»، «كلمة المرور مطلوبة».
- [ ] **T2.3** **إرسال:** `aria-busy` على الزر أو نص «جاري تسجيل الدخول…» + `disabled` لمنع الضعف.
- [ ] **T2.4** **401/شبكة:** «البريد أو كلمة المرور غير صحيحة» (بدل تفصيل 401) / «تعذّر الاتصال بالخادم».
- [ ] **T2.5** **رابط** «نسيت كلمة المرور؟» ب`text-sm` تحت الزر.

### 5.2 التسجيل — **إلزامي**

- [ ] **T2.6** **طول النموذج:** إن > 6 حقول — فكّر بخطوتين (UX فقط) مع **شريط تقدم** داخلي.
- [ ] **T2.7** **كلمة مرور:** قوة/تطابق (إن وُجد) — تلميحات **قصيرة** تحت الحقل.
- [ ] **T2.8** `autocomplete="email"` / `new-password` حسب الحقل.

### 5.3 نسيت كلمة المرور — **مهم**

- [ ] **T2.9** **بعد الإرسال:** «إن وُجد حساب مرتبط، ستصلك تعليمات…» (صياغة دون وعد مطلق إن كان يسبب مشكلة أمنية — اضبط مع المنتج).
- [ ] **T2.10** **حقول** بدون `placeholder` فقط — دائمًا `Label`.

### 5.4 Onboarding — **إلزامي**

- [ ] **T2.11** **Stepper:** نص: «الخطوة 2 من 3: بيانات المخزن».
- [ ] **T2.12** **رجوع:** يحافظ على المدخلات إن كانت `client` state؛ إن **لا** — نافذة «سيتم تجاهل التغييرات».
- [ ] **T2.13** **sample-data:** فقرة قصيرة: ماذا يُضاف (عينة أصناف/عملاء) — **وصف UX** فقط.

### 5.5 انتهاء الاشتراك / الفوترة — **مهم**

- [ ] **T2.14** `billing/expired`: عناوين: «انتهى اشتراكك» + «للمتابعة، راسِنا / حدّث خطتك» (حسب تطبيقك اليدوي).
- [ ] **T2.15** `billing` الرئيسي: **بطاقة** الخطة (اسم + حدود) — `outline` واضح، CTA `primary` واحد.
- [ ] **T2.16** `billing/history`: جدول بسيط — تاريخ، مبلغ، حالة — empty state: «لا توجد فواتير فوترة بعد».

## 6. نسخ جاهز

| الحالة | نص |
|--------|-----|
| حقل فاضٍ | «هذا الحقل مطلوب» |
| بريد غير صالٍ | «تأكد من صيغة البريد الإلكتروني» |
| تسجيل دخول فاشل | «بيانات الدخول غير صحيحة» |
| نجاح تسجيل | «تم إنشاء الحساب — يمكنك تسجيل الدخول» |
| فشل عام | «تعذّر إكمال الطلب. حاول مرة أخرى» |

## 7. نعم / لا

| نعم | لا |
|-----|-----|
| عرض نموذج بعرض `max-w-md` | نموذج بعرض كامل على شاشة واسعة |
| خطأ تحت الحقل | toast وحده للحقول |
| Onboarding بخطوات واضحة | 10 صفحات دون مؤشر تقدم |
