# خطة UI/UX — الخزينة والمالية (Finance / Treasury)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **أرقام واضحة**، **ألوان بمعنى واحد (لا تضليل)**: الأحمر/الأخضر **فقط** لدلالات يوافق عليها المنتج.

---

## 1. نظرة عامة

| المسار | محتوى |
|--------|--------|
| `.../finance` | ملخص/روابط (إن وُجد) |
| `.../finance/treasury` أو `treasuries` | رصيد وحركات |
| `.../receipts/new`, `.../payments/new` | سندات قبض/صرف |
| `.../expenses` | مصروفات |

**مكوّنات:** `TreasuryList` / `Table`, `PaymentReceiptForm`, `VoucherForm`, `ExpenseForm` — أسماء تقريبية.

---

## 2. فلسفة «احترافي + بسيط»

- **صفحة مالية = رقمان أو ثلاثة بَارزون** في الأعلى (رصيد/مدفوع/مستحق حسب المنتج) — **لا** 20 بطاقة.
- **سند = نموذج قصير:** طرف + مبلغ + خزينة + تاريخ + ملاحظة — **لا** حقول اختيارية بلا `Label`.
- **أي «مدين/دائن»** في الواجهة: عربي واضح — لا مصطلحات محاسبية دون تلميح.

---

## 3. تدفقات

1. **رصيد الخزينة** → **حركة** → (تفصيل) — من الأعمق للأعمق.
2. **سند قبض:** اختيار عميل/مصدر + مبلغ → حفظ → toast + رجوع أو إبقاء لطباعة (إن وُجد).
3. **صرف/مصروف:** مماثل + فئة (إن وُجد) — **تأكيد** عند مبالغ كبيرة (قرار product).

---

## 4. الأهداف

| # | الهدف | مؤشر |
|---|--------|------|
| G1 | أرقام قابلة المس | `tabular-nums` + فواصل عملة |
| G2 | جدول حركات usable | تمرير/فرز/تاريخ |
| G3 | نماذج مالياً | تحقق + منع double submit |
| G4 | لا تضليل بصري | لون = معنى واحد عبر الموديولات |

## 5. التسكات (مفصّلة)

### 5.1 الخزينة / الأرصدة — **إلزامي**

- [ ] **T8.1** **بطاقة رصيد** لكل خزينة (أو **صف** رئيسي): **اسم** + **رقم** — الرقم `text-2xl font-bold` `tabular-nums`.
- [ ] **T8.2** **حركات:** جدول: تاريخ، البيان، مدين، دائن، رصيد (إن وُجد) — **لا** ألوان فوضوية لكل سطر.
- [ ] **T8.3** **فلتر** تاريخ — نفس **نمط** التقارير (`ReportFilters` إن وُجد) للاتساق.
- [ ] **T8.4** **فارغ:** «لا حركات في الفترة» — ليس «لا data».

### 5.2 سندات القبض/الصرف — **إلزامي**

- [ ] **T8.5** **مبلغ:** `input` رقمي + `dir="ltr"` **لحقل المبلغ فقط** إن تحسن القراءة (— **قرار** اتجاه: يُوثَق)؛ **أو** إبقاء عربي مع `tabular-nums`.
- [ ] **T8.6** **خزينة:** `Select` واضح — افتراضي **يُظهر** كقيمة لا كفراغ.
- [ ] **T8.7** **طرف** (عميل/مورد): بحث combobox.
- [ ] **T8.8** **حفظ:** «جاري التسجيل…» + تعطيل.

### 5.3 القسائم (Vouchers) — **مهم**

- [ ] **T8.9** **تمييز مدين/دائن:** عمودان أو **شارتين** صغيرتين **لا** يلتبسان — ألوان `muted` للخلفية، **ليس** `destructive` لكل سطر.
- [ ] **T8.10** **ملخّص** أسفل النموذج: يطابق **المجاميع** إن وُجدت (عرض واجهي).

### 5.4 المصروفات — **مهم**

- [ ] **T8.11** **قائمة:** فئة + مبلغ + تاريخ + ملاحظة مختصرة.
- [ ] **T8.12** **نموذج جديد:** فئة **إلزامية** إن طلبها المنتج.
