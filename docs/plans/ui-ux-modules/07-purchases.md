# خطة UI/UX — المشتريات (Purchases)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **انسجام مع المبيعات** مع **لغة مميّزة** (مورد / شراء) حتى لا يخلط المستخدم فاتورتين.

---

## 1. نظرة عامة

| المسار | محتوى |
|--------|--------|
| `/dashboard/purchases` | مدخل (إن وُجد) |
| `.../purchases/invoices`, `/new` | فواتير وشراء جديد |
| `.../orders` | أوامر شراء (إن وُجد) |
| `.../returns` | مرتجعات شراء (إن وُجد) |

**الهدف الذهني:** نفس **بنية** فواتير المبيعات (جدول + بنود + حالة) لكن **عناوين** و**شارات** تعكس «شراء» وليس «بيع».

---

## 2. فلسفة «احترافي + بسيط»

- **لا تخترع نمط جدول جديد** — انسخ الأنماط من `06-sales` (عرض/فلاتر) مع تغيير **النصوص** فقط.
- **وضوح المورد** كما وضوح «العميل» في المبيعات: **دائمًا** حقل/عمود واضح.

---

## 3. تدفقات

1. **قائمة مشتريات** → **فاتورة** → (طباعة إن وُجدت).
2. **أمر شراء** → (استلام/تحويل لـ فاتورة؟) — **وصف** للمستخدم إن وُجد في المنتج.
3. **مرتجع شراء** → تأكيد + نجاح + رجوع — **نص** يذكر **أثر المخزون** إن عرضتموه (UX copy فقط).

---

## 4. الأهداف

| # | الهدف | مؤشر |
|---|--------|------|
| G1 | اتساق UI مع المبيعات | نفس مكوّنات الجدول/النماذج حسب الممكن |
| G2 | تمييز بصري خفيف | لون/أيقونة/عنوان «مشتريات» — بلا مبالغة |
| G3 | مورد بارز | عمود/بطاقة دائمًا |
| G4 | مرتجعات | تأكيد + نجاح واضح |

---

## 5. التسكات (مفصّلة)

### 5.1 القوائم — **إلزامي**

- [ ] **T7.1** **نفس** أسلوب فلاتر `06` — لا تباين ينهك المستخدم بين التطبيقين.
- [ ] **T7.2** **عنوان صفحة:** «فواتير المشتريات» — `h1` واضح؛ ليس فقط «الفواتير».
- [ ] **T7.3** **شارات:** مسميات عربية (مستلمة/مسودة… حسب الـ product).

### 5.2 فاتورة شراء جديدة — **إلزامي**

- [ ] **T7.4** اختيار **مورد** قبل/مع البنود — إن **إلزامي**: منع الحفظ مع «اختر موردًا».
- [ ] **T7.5** **بنود:** نفس **جدول** بندوات المبيعات؛ أسماء أعمدة عربية.

### 5.3 أوامر الشراء — **مهم**

- [ ] **T7.6** **دورة الحالة** (إن وُجدت): **شريط** بسيط أو **Badges** مرتبة زمنيًا.
- [ ] **T7.7** **فارغ:** «لا أوامر شراء بعد» + CTA.

### 5.4 المرتجعات — **مهم**

- [ ] **T7.8** `AlertDialog` عند «تأكيد المرتجع» — «لن تتمكن من التراجع تلقائيًا…» (حسب المنتج).
- [ ] **T7.9** **بعد النجاح:** «تم تسجيل المرتجع» + رجوع لقائمة.

---

## 6. نسخ جاهز

| الحالة | نص |
|--------|-----|
| بدون مورد | «اختر المورد أولًا» |
| فاوت شراء | عنوان «فاتورة شراء #…» (وليس مبيعات) |

---

## 7. نعم / لا

| نعم | لا |
|-----|-----|
| تكرار نمط `sales` | جدول مختلف بلا مبرر |
| عنوان مُصرَّح «مشتريات» | عناوين عامة مبهمة |

---

- `docs/ui_ux_audit.md` — قسم Purchases.
