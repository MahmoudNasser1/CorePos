# خطة UI/UX — نقطة البيع (POS)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **أقصى بساطة، أقصى سرعة**: عنصر واحد يسيطر على الشاشة لكل لحظة (بحث **أو** سلة **أو** دفع).

> **الأولوية القصوى** لجودة UI/UX في المشروع. أي تحسين هنا يسبق «زينة» باقي الصفحات.

---

## 1. نظرة عامة

**المسار:** `src/app/(dashboard)/dashboard/pos/page.tsx`  
**مكوّنات:** `POSProductGrid`, `POSCart`, `PaymentModal`, `CustomerSelect`, `POSReceipt`, `HeldCartsModal`, `useBarcodeScanner`, هيدر POS.

**قاعدة المستخدم:** يريد «بيع خلال ثوانٍ» — **لا** مقدمات طويلة، **لا** فتح **5** modals تسلسلية بدون داعٍ.

---

## 2. فلسفة «احترافي + بسيط» (POS)

| المبدأ | التطبيق |
|--------|---------|
| **معلومة واحدة بَارِزة** | الإجمالي + زر «الدفع» — لا تتنافس 3 مبالغ بنفس `text-2xl` |
| **نفس المكان دائمًا** | البحث أعلى الشبكة؛ السلة دائمًا على نفس الجهة (يمين/يسار) حسب تخطيطكم |
| **تغذية بصرية فورية** | تغيير الكمية = تحديث السطر + الإجمالي **بدون** وميض بأكمله |
| **الخطأ = قابل للتصحيح** | كمية غير صالحة → **تحت** الحقل أو `toast` قصير + بقاء التركيز |
| **الباركود ليس «سحر»** | عند فشل القراءة: «جرّب البحث بالاسم» — جملة واحدة |

---

## 3. تدفقات حرجة (User journeys)

1. **بحث → إضافة → دفع (نقد):**  
   تركيز في البحث → نقرة صنف → تظهر في السلة → «الدفع» → تأكيد مبلغ → **نجاح** → رقم فاتورة.
2. **باركود:**  
   بيب من القارئ → **نفس** نتيجة البحث عند تطابق — لا يحتاج المستخدم ينتقل بصرًا.
3. **تعليق / استئنف سلة:**  
   حفظ سلة → قائمة **معلّقة** بوقت واضح → استئناف.
4. **عميل اختياري/إلزامي** حسب المنتج:  
   اختيار واضح؛ إن إلزامي — امنع الدفع مع **رسالة** قصيرة بالعربية.

---

## 4. الأهداف (الجدول + تفصيل)

| # | الهدف | مؤشر نجاح | ملاحظة |
|---|--------|------------|--------|
| G1 | تفاعل سلة | تحديث إجمالي **بلا تأخير ملحوظ** | قياس بصري + DevTools إن لزم |
| G2 | منع دفع مزدوج | أول نقرة: `disabled` + «جاري التسجيل…» | **Gate إلزامي** |
| G3 | نجاح بيع | رقم فاتورة + طباعة + بيع جديد | لا يُفقد الرقم |
| G4 | بحث/شبكة | تغذية مباشرة للشبكة؛ **loading** بسيط للشبكة | |
| G5 | RTL + أرقام | `+`/`−` بجانب الكمية بلا انعكاس مربك | `tabular-nums` |
| G6 | إيصال 80mm | معاية طباعة: مبلغ/اسم/رقم/تاريخ **غير** مقطوع | |

## 5. التسكات (مفصّلة)

### 5.1 الشبكة والبحث — **إلزامي**

- [ ] **T4.1** **شريط بحث:** `placeholder` عربي: «ابحث بالاسم أو الباركود» — **ليس** «Search…».
- [ ] **T4.2** **نتائج:** highlight الجزء المطابق (نص) إن تيسّر.
- [ ] **T4.3** **صور:** `placeholder` صورة منتج (أيقونة صندوق) **متسق** — لا ارتفاعات عشوائية.
- [ ] **T4.4** **نفاد المخزون:** بطاقة **مُخففة** `opacity-60` + شارة «غير متوفر» + **تعطيل** النقرة (أو نقرة = توضيح) — اختر **سلوكًا واحدًا** ووثّق.
- [ ] **T4.5** **شبكة responsive:** 2–4 أعمدة حسب العرض — لا بطاقة ضيقة بلا `min-h`.

### 5.2 الباركود — **إلزامي**

- [ ] **T4.6** **Hook:** اختبر مع إدخال سريع (محاكاة) — لا يتداخل مع حقل البحث إن كان مفعّلًا في نفس الوقت.
- [ ] **T4.7** **فشل:** «لم يُعثر على المنتج — جرّب البحث» — دون **stack trace**.

### 5.3 السلة — **إلزامي**

- [ ] **T4.8** **فارغة:** «السلة فارغة — ابحث عن صنف أو استخدم الباركود» + أيقونة هادفة.
- [ ] **T4.9** **± والكمية:** `min=1`؛ عند `max` من المخزن — **منع** الزيادة + `toast` قصير «الكمية المتاحة: X».
- [ ] **T4.10** **إجمالي:** سطر `font-semibold` + عملة `formatCurrency`؛ `tabular-nums` للأرقام.
- [ ] **T4.11** **إزالة سطر:** أيقونة سلة/× + **تأكيد اختياري** فقط للسلة الكبيرة (إن لزم — لا تزيد الاحتكاك في المحل).

### 5.4 العميل — **إلزامي (حسب المنتج)**

- [ ] **T4.12** `CustomerSelect`: حقل **واحد** واضح؛ بحث `debounce` ~200ms.
- [ ] **T4.13** **إضافة عميل سريع** (إن وُجد): نموذج **صغير** (Sheet) — لا 15 حقلًا.

### 5.5 الدفع — **إلزامي**

- [ ] **T4.14** **Dialog:** `DialogTitle` عربي: «دفع الفاتورة» + المبلغ بخط أوضح من النص.
- [ ] **T4.15** **طرق الدفع:** أزرار **متساوية** العرض في صف أو شبكة 2 — نقد / آجل / … — أيقونات + نص.
- [ ] **T4.16** **Double-submit:** `pending` + `aria-busy` + تعطيل «تأكيد» و«إلغاء» (أو فقط تأكيد) — **ممنوع** نقر ثانٍ يسجّل عملية.
- [ ] **T4.17** **خطأ دفع:** «تعذّر تسجيل البيع. تحقق من الاتصال» + «إعادة المحاولة».

### 5.6 ما بعد البيع — **إلزامي**

- [ ] **T4.18** **شاشة/حالة نجاح:** **رقم الفاتورة** `text-2xl` + أزرار: «طباعة»، «فاتورة جديدة» (primary/secondary بترتيب منطقي).
- [ ] **T4.19** **لا** تُغلق النجاح تلقائيًا بسرعة تمنع نسخ الرقم (إن كان المستخدم يحتاجه).

### 5.7 الإيصال والطباعة — **إلزامي**

- [ ] **T4.20** `POSReceipt`: عرض 80mm في DOM بنسبة **ضيّقة** (`max-w-[80mm]`) + خط monospace للأرقام إن لزم.
- [ ] **T4.21** قاعدة `@media print`: إخفاء باقي الـ app؛ **فقط** الإيصال.
- [ ] **T4.22** اختبر **هوامش** طابعة حراري — لا قص لاسم الشركة/الإجمالي/الرقم.

### 5.8 السلات المعلقة — **مهم**

- [ ] **T4.23** `HeldCartsModal`: كل عنصر: وقت، مبلغ تقريبي، «استئناف» / «حذف» مع `AlertDialog` للحذف.
- [ ] **T4.24** **حذف:** «سيتم حذف السلة نهائيًا» — ليس فقط «موافق».

### 5.9 اختصارات — **مهم**

- [ ] **T4.25** تلميح صغير في زاوية: «اختصارات: …» — فقط **إن** مفعّلة في الكود.
- [ ] **T4.26** **لا** تتعارض اختصارات مع إدخال عربي/IME (تحقق عملي).

### 5.10 الهيدر (فرع/مخزن) — **مهم**

- [ ] **T4.27** اختيار **واحد** واضح؛ القيم الافتراضية من الـ API تُعرض **بسلاسة** (لا اختيار وهمي) — handoff عند لبس منطق.

## 6. نسخ جاهز

| الحالة | نص |
|--------|-----|
| سلة فارغة | «السلة فارغة. ابحث عن منتج أو امسح الباركود.» |
| غير متوفر | «هذا المنتج غير متوفر حاليًا.» |
| دفع جارٍ | «جاري تسجيل البيع…» |
| نجاح | «تمت عملية البيع بنجاح» + رقم الفاتورة |
| فشل شبكة | «تعذّر الاتصال. جرّب مرة أخرى.» |

---

## 7. نعم / لا

| نعم | لا |
|-----|-----|
| زر دفع رئيسي واحد بنفس لون primary دائمًا | عدة «احفظ/أكد/ادفع» بنفس الوزن |
| معاية إيصال قبل `print` | طباعة مباشرة بلا رؤية |
| تعطيل أثناء الإرسال | نقر مزدوج يسبب فواتير مكررة |
