# خطة UI/UX — المساعدة + سجلات التدقيق + سوبر أدمن (Help / Audit / Super-admin)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **قراءة ومسح (scan)** أوضح من «زخرفة»؛ **سجلات = كثافة معلومة** — يجب **ترقيم/بحث/فلتر** لا جدار نص.

---

## 1. نظرة عامة

| المسار | غرض |
|--------|------|
| `/dashboard/help` | مركز مساعدة / أسئلة |
| `/dashboard/audit-logs` | سجلات تدقيق (من فعل ماذا / متى) |
| `/super-admin` (إن وُجد) | شاشة إدارة داخلية |

**مخاطر UX:** **محتوى** placeholder يبني **عدم ثقة** — الأفضل بضع **مقالات حقيقية** قصيرة + «تواصل معنا» من لافتة «قريبًا» طويلة.

---

## 2. فلسفة «احترافي + بسيط»

- **المساعدة:** مقالات `Card` + **عناوين h2** قابلة للقفز — **موسّع قابل للطي** (Accordion) يقلل الارتفاع.
- **السجلات:** جدول **كثيف** لكن **مقروء** — `text-sm` + `font-mono` **فقط** لـ IDs/طلب.
- **سوبر أدمن:** **تحذير** بصري `border-amber-500/40` (أو ما يوازيها) + نص: «للمسؤولين فقط».

---

## 3. تدفقات

1. **مساعدة** → **بحث** (إن وُجد) → **مقالة** / **رجوع** للفهرس.
2. **سجلات** → **فلتر** تاريخ/مستخدم/نوع → **صف** → (تفصيل) — **تصدير** (إن وُجد).
3. **سوبر أدمن** → **إجراء** خطير → `AlertDialog` **ثنائي** (UX فقط — لا تغيير policy).

---

## 4. الأهداف

| # | الهدف | مؤشر |
|---|--------|------|
| G1 | مساعدة مفيدة | 5+ عناوين حقيقية أو رابط دعم |
| G2 | سجلات قابلة للعثور | بحث/فلتر/ترقيم |
| G3 | سوبر أدمن | تحذير + تأكيدات |
| G4 | A11y | عناوين واضحة، جدول بـ `scope` (إن HTML خام) |

---

## 5. التسكات (مفصّلة)

### 5.1 المساعدة — **مهم**

- [ ] **T12.1** **هيكل:** `h1` «مركز المساعدة» + `Input` بحث (إن وُجد) placeholder «ابحث في المساعدة…».
- [ ] **T12.2** **أقسام:** مثال: **POS** / **مخزون** / **فواتير** / **مشاكل شائعة** — ليس «قسم 1».
- [ ] **T12.3** **مقال:** `prose` (إن وُفّر) + قوائم **مرقّمة** لخطوات.
- [ ] **T12.4** **لا** صفحات **فارغة** — **إن** المحتوى ناقص: بطاقة «نُحدّث المحتوى قريبًا» + بريد/واتساب (حسب product).
- [ ] **T12.5** **روابط خارجية:** `target="_blank"` + `rel` + نص: «يُفتح في نافذة جديدة».

### 5.2 سجلات التدقيق — **مهم**

- [ ] **T12.6** **جدول:** أعمدة: الوقت، المستخدم/الوكيل، الفعل، الكيان، تفاصيل/معرف.
- [ ] **T12.7** `requestId` / معرف: **نسخ** بزر `Copy` (إن وُجد) + `toast` «تم النسخ».
- [ ] **T12.8** **فلتر** تاريخ — نفس `ReportFilters` إن ممكن (اتساق).
- [ ] **T12.9** **تصدير** — `loading` + عدم **تجمد** الشاشة.
- [ ] **T12.10** `empty` — «لا سجلات في الفترة».

### 5.3 سوبر أدمن — **حسب التوفر**

- [ ] **T12.11** **لافتة** واضحة أعلى الصفحة (غير dismissing إن كان ضروريًا أمنيًا).
- [ ] **T12.12** **إجراءات خطرة:** نص + تأكيد + إدخال نص (إن product يطلبه) — **UI فقط**؛ لا اختراع policy.
- [ ] **T12.13** **لا** **تنقل** `super-admin` داخل `Sidebar` **العام** — أو مع **icon** مميّز + صلاحية (handoff product).

---

## 6. نسخ جاهز

| الحالة | نص |
|--------|-----|
| لا نتائج بحث مساعدة | «لا نتائج — جرّب كلمات أخرى أو تواصل مع الدعم» |
| سجلات | «سجل التدقيق يساعدك على تتبع التغييرات» (وصف `text-sm` تحت `h1`) |
| نسخ معرف | «تم نسخ المعرف» |

---

## 7. نعم / لا

| نعم | لا |
|-----|-----|
| مقالات قصيرة قابلة المس | جدران نص 40 سطر بلا فهرس |
| سجلات بجدول | JSON خام للمستخدم |

## 8. معايير القبول

- `help` **لا** تترك **كليًا** بلا CTA/محتوى.
- `audit-logs` **قابلة للاستعمال** (بحث/فلتر).
- `docs/ui_ux_audit` — أقسام Help/Audit/Super-admin.
