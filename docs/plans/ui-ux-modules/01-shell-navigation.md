# خطة UI/UX — الهيكل العام والتنقل (Shell & Navigation)

**المرتكز:** [00-design-principles.md](./00-design-principles.md) — **عنوان واحد بصريًا** أعلى المحتوى، **تنقل ثابت** لا يسرق تركيز POS/الفواتير.

---

## 1. نظرة عامة

| العنصر | دوره |
|--------|------|
| Sidebar | تنقل بين الأقسام الرئيسية، حالة نشطة |
| Header | مستخدم، إشعارات (إن وُجدت)، مساعدة/خروج |
| المحتوى | `max-w` عند الحاجة؛ `space-y` بين الأقسام |
| Toaster | تغذية راجعة قصيرة، RTL |
| Command Menu | اختصار عام (إن وُجد) |

**ملفات مرجعية:** `src/app/layout.tsx`, `src/components/layout/Sidebar.tsx`, `Header.tsx`, `error.tsx`, `loading.tsx`, `src/components/ui/sonner`, `CommandMenu` إن وُجد.

---

## 2. فلسفة «احترافي + بسيط» لهذا الموديول

- **Sidebar = فهرس، ليس لوحة معلومات:** أسماء قصيرة، ترتيب من «الأكثر يومية» (POS) إلى «الأقل» (إعدادات) حيث ينطبق.
- **لا تزيد عدد الألوان في القائمة:** عنصر نشط = **خلفية** أو **شريط جانبي** واحد واضح، لا 3 درجات مختلفة لنفس المعنى.
- **الهيدر رقيق:** لا يتصارع مع عنوان الصفحة؛ الارتفاع ثابت تقريبًا عبر الصفحات.

---

## 3. تدفقات حرجة (يجب أن تمر سلسًا)

1. **أول دخول للوحة:** يظهر Sidebar + محتوى **بدون** وميض إعادة ترتيب عناصر.
2. **التنقل من POS إلى المخزون والعودة:** لا تفقد السياق البصري (scroll position اختياري — المهم عدم «قفز» مفاجئ بدون تحميل).
3. **خطأ شبكة عام:** شريط أو حالة في `error.tsx` + **إعادة محاولة** — لا صفحة بيضاء.

---

## 4. الأهداف

| # | الهدف | مؤشر نجاح |
|---|--------|-----------|
| G1 | وصول ≤ 3 نقرات لأي قسم رئيسي من الصفحة الرئيسية | مسار واضح في الـ Sidebar |
| G2 | RTL في القوائم المنسدلة | محاذاة `end` للـ popover حيث يلزم |
| G3 | تحميل/خطأ مفهوم | skeleton أو رسالة عربية + إجراء |
| G4 | Toaster متسق | `dir="rtl"`، مدة مناسبة، لا تكدس 5 toasts دفعة واحدة |
| G5 | A11y للأيقونات | `aria-label` لأزرار «…» فقط |

## 5. التسكات (مع تفاصيل فرعية)

### 5.1 الشريط الجانبي — **إلزامي**

- [ ] **T1.1** **تجميع:** إن كان ≥ 12 رابطًا: قسّم إلى أقسام بعناوين فرعية صغيرة `text-xs text-muted-foreground` (مثال: «المبيعات والمخزون»، «الحسابات») — **بدون** إغراق في عناوين فرعية كثيرة.
- [ ] **T1.2** **Active state:** خلفية `bg-accent` أو شريط `border-s-2 border-primary` — **واحد** فقط كنمط رئيسي.
- [ ] **T1.3** **Scroll:** إن طال العمود: `overflow-y-auto` + `pb` كافٍ حتى لا يختفي آخر رابط خلف شريط النظام.
- [ ] **T1.4** **موبايل:** في الشاشات الأصغر من `md` — درج (`Sheet`) يفتح بزر واضح؛ الزر في **قاعدة إبهام** (أسفل يمين/يسار حسب التصميم) أو أعلى الهيدر — **وثّق** الاختيار في `ui_ux_audit`.

### 5.2 الرأس — **إلزامي**

- [ ] **T1.5** **مستخدم:** اسم مختصر + قائمة (الخروج، الإعدادات إن وُجدت) — لا قائمة بدون تسمية.
- [ ] **T1.6** **زحمة:** في العرض الضيق، اضغط العناصر إلى `…` واحد بدل 4 أيقونات منفصلة.
- [ ] **T1.7** **Breadcrumbs:** إن وُجدت — `text-sm`، فاصل `›` أو `/` مع `dir` صحيح للنصوص المختلطة (نادر).

### 5.3 عناوين الصفحات — **مهم**

- [ ] **T1.8** **نمط موحّد:** كل `page.tsx` تحت `/dashboard` يبدأ بـ:
  - `h1` واحد (`text-2xl` أو `text-3xl`) + **سطر مساعد اختياري** `text-muted-foreground` جملة واحدة.
- [ ] **T1.9** **إجراء رئيسي في الصفحة:** يظهر في **زاوية** بمحاذاة العنوان (stack على الموبايل: عنوان فوق، زر تحت بعرض كامل).

### 5.4 تحميل وأخطاء — **إلزامي**

- [ ] **T1.10** **loading.tsx:** إن وُجد — skeleton لبطاقات مماثلة للوحة أو شريط تحميل علوي **خفيف**.
- [ ] **T1.11** **error.tsx:** عنوان «تعذّر التحميل» + وصف سطر + زر «إعادة المحاولة».

### 5.5 الإشعارات و Command — **مهم**

- [ ] **T1.12** **Toaster:** تأكد من `richColors` لا تُربك (نجاح أخضر هادئ — ليس صارخًا لكل نقرة).
- [ ] **T1.13** **Command:** إن وُجد — placeholder عربي، نتائج مقسّمة (صفحات / إجراءات)، **Esc** للإغلاق.

### 5.6 الوضع الداكن — **polish**

- [ ] **T1.14** **تباين:** حدود `border-border` واضحة؛ نص `muted` لا يقل عن ~4.5:1 إن أمكن (تقدير بصري + DevTools).

## 6. نسخ جاهز (Microcopy)

| السياق | نص مقترح |
|--------|-----------|
| خطأ عام | «تعذّر تحميل الصفحة. تحقق من الاتصال ثم أعد المحاولة.» |
| إعادة المحاولة | «إعادة المحاولة» |
| تحميل | «جاري التحميل…» (يُفضّل skeleton على جملة طويلة) |
| قائمة مستخدم | «الحساب» / «تسجيل الخروج» |

---

## 7. نعم / لا (سريع)

| نعم | لا |
|-----|-----|
| قوائم مجمّعة عند كثرة الروابط | ألوان عشوائية لكل قسم |
| زر إجراء واحد بارز في الهيدر إن لزم | 3 أزرار primary في الشريط العلوي |
| أيقونات + نص في الـ Sidebar | أيقونات فقط بدون تلميح نصي قصير على سطح المكتب |

---

## 8. معايير قبول الموديول

- لا أزرار أيقونات بدون `aria-label`.
- لا نص إنجليزي في Sidebar/Header.
- توثيق 🔴/🟡 في `docs/ui_ux_audit.md`.
